# PLAN.md — Election Process Education

Single source of truth for scope, architecture, data, UI, and delivery.
All agents and contributors execute against this file.

## 1. Goal
An interactive assistant that explains the election process, timelines,
and steps to citizens, first-time voters, and students — grounded in live
official data, powered by Gemini on Vertex AI, deployed to Cloud Run.

## 2. Mandatory Constraints
- Built inside Google Antigravity.
- Deployed to Google Cloud Run (URL is the live preview).
- Uses Vertex AI for Gemini access.
- Public GitHub repo, single `main` branch, < 1 MB total.
- No bundled datasets; all election content fetched at runtime.

## 3. Users & Core Jobs
- First-time voter: "Am I eligible and how do I register?"
- Student: "Explain the full election process with a diagram and quiz."
- Citizen: "What phase is the current election in and when do I vote?"

## 4. Feature Set (v1)
- [ ] Conversational chat with streaming Gemini responses and citations
- [ ] Interactive timeline by country/state/election-type
- [ ] Six learn modules: Registration, Nomination, Campaigning,
      Polling Day, Counting, Results
- [ ] On-demand quizzes generated by Gemini
- [ ] Glossary with Gemini-backed definitions
- [ ] Eligibility & registration wizard
- [ ] Polling station lookup (address or geolocation)
- [ ] Sources/transparency page with live health status
- [ ] EN + HI i18n via on-the-fly Gemini translation with cache
- [ ] Light/dark themes, WCAG AA

## 5. Architecture
Single Cloud Run container:
- FastAPI serves `/api/*` and static React build at `/`.
- Vertex AI SDK for Gemini.
- httpx connectors for each upstream source.
- In-memory TTL cache.
- Secret Manager for credentials.

## 6. Data Plan (API-driven only)
| Source | Purpose | Auth |
|---|---|---|
| ECI open data & Voter Helpline | Schedules, candidates, stations | None |
| NVSP | Registration flow, EPIC status | None |
| Google Civic Information API | US polling + reps | GCP key |
| data.gov.in CKAN | Historical turnout, metadata | None |
| OpenStates / Ballotpedia public endpoints | US state procedures | None |
| Wikipedia REST | Glossary fallback | None |

Normalized internal shapes:
- `TimelineItem { phase, start, end, legal_ref, source_url }`
- `Step { order, title, description_md, who, source_url }`
- `GlossaryTerm { term, definition_md, source_url }`

Cache TTLs: schedules 1h, procedural 24h, glossary 7d.

## 7. API Surface
- `GET  /api/health`
- `POST /api/chat`              body: { message, session_id, locale }
- `GET  /api/timeline`          ?country&state&election_type
- `GET  /api/steps/{topic}`
- `GET  /api/glossary/{term}`
- `POST /api/quiz`              body: { topic, difficulty, count }
- `GET  /api/polling-station`   ?address | ?lat&lng
- `GET  /api/sources`

## 8. Gemini Usage
- Flash: intent classification (JSON mode), translation.
- Pro: final grounded answers, quiz generation, long explanations.
- System prompt enforces: answer only from provided context, cite
  source URL, refuse if context empty.

## 11. Deployment
Multi-stage Dockerfile:
1. `node:20-alpine` builds Vite app → `/web/dist`.
2. `python:3.11-slim` installs deps, copies `/web/dist` to `/app/static`.
3. FastAPI mounts static at `/`.

Deploy command:

```cmd
gcloud run deploy election-edu
--source . 
--region asia-south1 
--allow-unauthenticated 
--set-env-vars VERTEX_PROJECT=$PROJECT,VERTEX_LOCATION=us-central1 
--set-secrets GEMINI_SA_KEY=gemini-sa:latest
```


## 12. CI/CD

## 14. Risks & Mitigations
- Upstream API schema drift → connector isolation + contract tests.
- Rate limits → TTL cache + exponential backoff.
- Hallucinated facts → strict RAG prompt + "no source → refuse" rule.
- Prompt injection from fetched HTML → sanitize + UNTRUSTED wrapping.
- Repo size creep → pre-commit `du -sh .` hook + CI gate.
- Cost → min-instances 0, Gemini Flash for classification.
